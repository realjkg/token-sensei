Bede is a Socratic AI tutor that works the way Charlotte Mason intended — through living ideas, narration, short lessons, and wonder. One parent configures the daily plan; each child logs in from their own tablet. Bede opens every subject with a question, not a lecture, and celebrates discovery rather than correct answers.

What it does
Six capabilities in one tutor
Charlotte Mason Subjects
Morning Time, Living Books, Nature Study, History, Language Arts, Mathematics, Science, Saints & Catechism, Art & Music, and Free Study — each with grade-appropriate durations.
Socratic Dialogue
Bede never lectures. It opens with a question, offers hints when a child is stuck, and celebrates the moment of discovery — building reasoning over recall.
Handwriting Canvas
Children draw or write their answers with an Apple Pencil, Livescribe pen, or any stylus. The sketch is sent directly into the conversation.
Text-to-Speech
Bede's responses are spoken aloud via browser TTS — ideal for K–3 readers who are still developing fluency and for read-aloud sessions.
Session Summary
At the end of each session Bede generates a concise parent report: what was covered, how the child engaged, and what to revisit next time.
Faith Integration
Every subject includes optional connections to saints, scripture, or the theological virtues — woven naturally, never forced, through the connect_to_faith tool.
Setup
Up and running in four steps
1
Run make setup
Generates .env, starts Caddy, nginx, and FastAPI via Docker Compose on your home server.
2
Install the CA cert
Run make caddy-trust on each tablet once — enables LAN HTTPS without browser warnings.
3
Configure students
Parent logs in, adds up to 10 students, selects subjects, grade, and context. Takes under five minutes.
4
Copy tablet link
Each student gets a /session?student=Name URL. Paste it in the tablet browser and they are ready to learn.
The Charlotte Mason Difference
Charlotte Mason taught that children are persons, not empty vessels. Bede reflects this: lessons are short (20 minutes for K–3, 60-minute blocks for grades 4–8), narration replaces tests, and living books replace textbooks. The AI is designed to step back — to prompt wonder, not to deliver information.
Technical reference
Stack at a glance
Layer Technology Notes
AI Model claude-sonnet-4-6 Streaming, max_tokens 400. Prompt caching on the static Sage persona block.
Summary Model claude-haiku-4-5 Non-streaming, max_tokens 600. End-of-session parent report only.
Backend FastAPI asyncpg SSE streaming, async SQLAlchemy, Pydantic Settings with production validation.
Frontend React Vite TypeScript Zustand store (sessionStorage), Tailwind CSS, Web Speech API for voice input.
Reverse Proxy Caddy + nginx Caddy generates a local CA for LAN HTTPS. Tablets trust it once via make caddy-trust.
Database PostgreSQL All student data and voice profiles stored as AES-256-GCM BYTEA. DB provider never sees plaintext.
Voice Auth Resemblyzer + Whisper Speaker embeddings + MFCC similarity scoring. Children speak a passphrase; server verifies identity.
Security
AES-256-GCM at rest
MASTER_SECRET derives a KEK which wraps a DATA_KEY. Every BYTEA column is encrypted before storage.
JWT device fingerprint
Tokens are bound to IP + User-Agent at issuance. Replaying a token from a different device returns 401.
ExfiltrationGuard
Middleware strips SSE data: lines containing prompt-injection markers before they reach the client.
Hardened containers
sage container user has no shell. All containers run read_only: true, cap_drop: ALL, no-new-privileges.
